1. org-attach

1.1. 方便地添加附件链接

在 .emacs 里加一句 (setq org-attach-store-link-p 'attached) 即可，见此。

1.2. 在默认文件浏览器中打开org-attach的文件夹

用 (add-to-list 'org-file-apps '(directory . default)) 即可，类似这个问答。 还可以用字符串以及 %s ，见这个问答。

2. org-agenda

见手册。命令见这部分。

快捷键：

  • b 与 f 来前进与后退。
  • R 显示 Clock Table
  • l 打开 Logging。 C-u C-u l 只显示clock。

2.1. Agenda Files

C-c [ Add current file to the list of agenda files. C-c ] Remove current file from the list of agenda files. C-' C-, Cycle through agenda file list, visiting one file after the other.

2.2. 启动时打开org-agenda

(add-hook 'after-init-hook 'org-agenda-list) 即可，见此问答。

2.3. org-agenda显示clocked time

l （或 v l ，也即 org-agenda-log-mode ）即可。

或者用 R （即 clock-agenda-clockreport-mode ）即可。另外，可以用 org-agenda-clockreport-parameter-plist 来定义它的行为，见此问答或此问答。好像也可以用 org-clocktable-default ，见此问答。

2.4. & Window Split - (setq org-agenda-window-setup 'current-window)

Type C-h v org-agenda-window-setup for other options. https://stackoverflow.com/questions/10635989/emacs-org-agenda-list-destroy-my-windows-splits

2.5. 开多个org-agenda

用 M-x org-toggle-sticky-agenda 即可，或者在 Agenda Dispatcher 里按 * 。

3. org-export

详见手册。

3.1. 导出为Markdown

直接 C-c C-e ，在选择页面输入 m m 即可，见手册。

不过，可能需要用 M-x customize-option 先改一下 org-export-backends ，开启 Markdown 选项，见此问答。或者在.emacs里加一句 (require 'ox-md) ，见手册。

3.2. 只导出一个子树

用 C-c C-e C-s 即可，见此问答，以及手册。

3.3. 导出特定部分

把要导出的子树用 :export: 标记即可，见问答。

导出整个文档时，使用 #+EXCLUDE_TAGS: export 即可。

3.4. Error: Unable to resolve link

直接暴力改 org-export-with-broken-links 或者用 #+OPTIONS: broken-links:t 应该可以解决问题。见此issue。

3.5. beamer

文档是Beamer Export (The Org Manual)。这里是一个例子。教程见Writing Beamer presentations in org-mode或Using Beamer export to produce slideshows and article-style handouts from the same org source，Beamer export with Org-mode里还有更多。

但是，具体参数参考Export Settings (The Org Manual)以及Table of Contents (The Org Manual)。

文档里给出了一个例子：A Beamer example (The Org Manual)。

据说有一个 Minor Mode 的样子：Editing support (The Org Manual)。

  • Presentations with org-mode and beamer. | code, travel and explore.、
  • fniessen/refcard-org-beamer: Org Beamer reference card等。

麻烦的一点是 \pause 要用 #+BEAMER: \pause 才行……见Beamer specific syntax (The Org Manual)。

3.5.1. 显示不出section的名字

theme 问题，把主题从 default 改成 AnnArbor 之类的的应该就可以了。

3.5.2. 导出选项里没有beamer

需要自己加backend。

3.5.3. 长beamer

据说用 :BEAMER_opt: allowframebreaks,label= 即可，见org mode - autobreak/allowframebreaks/allowpagebreaks problem in long Org+Beamer file when exporting to long slides - TeX - LaTeX Stack Exchange

4.7. 显示图片

org-mode 似乎可以在行内显示本地图片，只需用 [[./img/cat.jpg]] 语法即可。用 C-c C-x C-v 开关，见手册。

4.8. 移动整个子树： M-UP 和 M-DOWN

M-UP 和 M-DOWN （移动一行是 M-S-UP 和 M-S-DOWN ）

4.10. folding可以不同的分屏： M-x clone-indirect-buffer

7. TODOs

7.1. TODO的优先级

见手册。似乎1~64都严格高于A，实验可得 A=65 B=66 C=67，以此类推。

如需要更多字母，如下更改即可，见此问答。 

(setq   org-enable-priority-commands t
    org-highest-priority ?A
    org-default-priority ?J
    org-lowest-priority ?J
)

7.2. 使用计时器

C-c C-x 0 开始正计时， C-c C-x ; 开始倒计时。 C-c C-x - 插入时间戳列表， M-RET 继续这个列表。 C-c C-x , 暂停计时器， C-c C-x _ 停止计时器。

7.3. Creating Timestamps, Deadlines and Scheduling

C-c . timestamp <> C-c ! inactive timestamp [] C-c C-d DEADLINE C-c C-s SCHEDULED

7.4. "[ ]" Items

https://orgmode.org/manual/Breaking-Down-Tasks.html

7.5. Closing Items

在.emacs里加一句 (setq org-log-done 'time) （或者 (setq org-log-done 'note) ）即可，见手册或指南。

7.6. 跟踪TODO

在.emacs里加 (setq org-todo-keywords '((sequence "TODO(t)" "WAIT(w@/!)" "|" "DONE(d!)" "CANCELED(c@)"))) 或者在文件里加 #+TODO: TODO(t) WAIT(w@/!) | DONE(d!) CANCELED(c@) 即可。 详见手册。

8. 时间戳

8.1. 加带时间戳的note

C-c C-z ，见手册。

8.2. 插入带小时和分钟的时间戳

在命令前加个 C-u 即可，见此回答。

8.3. 重复事件

Org本身有简单的语法支持，见Repeated tasks (The Org Manual)，但就如Org-mode Frequently Asked Questions所言，它局限性很大。

因此，直接 C-c C-x c （ M-x org-clone-subtree-with-time-shift ）复制整个子树并增加偏移即可，见UOMF: Recurring Events with Org Mode。

14. Column View

详见手册。

Agenda里见手册的这里。

在文件开头用 #+COLUMNS: %25ITEM %TAGS %PRIORITY %TODO 或者在根节点加个 :COLUMNS: 的属性即可。属性见手册这部分。

用 C-c C-x C-c 即可打开， e 编辑， q 退出， < 和 > 改变列宽度， S-M-RIGHT 增加新的一列，见这部分手册。

14.1. 时间戳

在 #+COLUMNS 里加一个 %TIMESTAMP_IA 即可。

14.2. 卡顿

就如 Large life-logging org-mode file gets too slow : orgmode 所说的一样，在大文件里（或 org-agenda 中）用 Column View 会卡顿。

有人建议用 #+STARTUP: showeverything ——虽然不知道有什么用。
